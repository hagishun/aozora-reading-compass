# 読後ログ テンプレート

青空文庫で読んだ作品を「点」ではなく「線」として残すための、最小のログ形式です。
1作品 = 1行 を基本にします。書くことのコストを限界まで下げて、続けやすさを優先します。

## 基本フォーマット (1行ログ)

```
- YYYY-MM-DD / 作品名 (作者) / state: <今日の状態> / 一行: <今日の自分に残った一行/感想>
```

### 例

```markdown
- 2026-05-03 / 夢十夜 (夏目漱石) / state: 静けさ / 一行: 「百年待っていてください」が今日の自分に効いた
- 2026-05-04 / 走れメロス (太宰治) / state: 刺激 / 一行: 信じることの重さを久しぶりに思い出した
- 2026-05-05 / 城の崎にて (志賀直哉) / state: 思考 / 一行: 死は隣にあるが、騒がしくない、という感触
```

## 使い方

1. 自分のリポジトリ or Notion or 任意のテキストファイルに `reading-log.md` を作る
2. 読んだ日に1行追記する
3. 月に1度くらい眺めて、自分が選びがちな state / theme の偏りを見る

## 拡張フィールド (任意)

書きたい日だけ書く、軽量な追加情報です。空欄でかまいません。

| フィールド | 例 | 役割 |
|---|---|---|
| `theme` | 自然 / 都市 / 死生観 | 推薦軸との対応を取りたいとき |
| `length_min` | 20 | 体感の読了時間 |
| `mood_after` | 整った / 重くなった / 笑った | 読後の気分変化 |
| `revisit` | yes / no | また読みたいか |

### 拡張フォーマット例

```markdown
- 2026-05-03 / 夢十夜 (夏目漱石)
  state: 静けさ
  theme: 夢, 死生観
  length_min: 30
  mood_after: 整った
  revisit: yes
  一行: 「百年待っていてください」が今日の自分に効いた
```

## 設計思想

- **書くハードルを上げない**: 100文字以内に収まる「一行」だけが必須
- **検索しやすい構造**: 行頭が日付、フィールドが固定なので grep / 集計が容易
- **自分の地図になる**: 続けることで「自分は静けさ系を選びがち」「思考系は溜まっている」など可視化される

書評ではなく **読書の航跡** を残すためのテンプレートです。
