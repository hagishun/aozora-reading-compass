# ADR-0001: 青空文庫の公開CSVをLLM推薦のgrounding sourceとする

- Status: Accepted
- Date: 2026-05-04
- 関連handbook原則: [llm-grounding](https://github.com/hagishun/hagishun-handbook/blob/main/principles/llm-grounding.md)

## Context

本プロジェクトの中核は「青空文庫の作品から、今日の自分に合う一冊をLLMに推薦させる」こと。素朴に LLM 単体で推薦させると、handbook の `llm-grounding` で整理した2つの限界に直撃する。

- **知識カバレッジの偏り**: LLM は教科書頻出の有名作（漱石「こころ」、芥川「羅生門」等）には強いが、約18,000ある青空文庫作品の長い裾野はほぼ知らない。結果として推薦は数十作の有名どころに収束する。
- **幻覚**: 「夏目漱石『秋の声』」のような実在しない作品名や、誤った作者名、捏造URLを返す。

「発見導線」というプロジェクトの目的（[README](../../README.md)）からすると、有名作偏重も嘘の作品も致命的。

## Decision

**青空文庫の公開CSV（`list_person_all_extended_utf8.csv` 等）から「実在する作品リスト」を構築し、LLM の出力をこのリストに照合してgroundingする。**

具体的には:

- リストには `id / title / author / aozora_url / char_count` 程度のメタデータのみを保持する
- 本文は取得しない（「やらないこと」を維持）
- 推薦時は LLM に自由に作品名を出させ、後段でリスト照合し、リスト外の出力は捨ててリトライする（"LLMに先に答えさせて後で照合" 方式）
- 手書きタグ（`states` / `themes`）は持たない。雰囲気・テーマへのマッチングは LLM の事前学習知識に任せる

## Alternatives

- **案A: LLM単独で推薦させる**
  - 却下理由: 有名作偏重と幻覚で「発見導線」の目的を満たせない
- **案B: 全作品のメタデータをプロンプトに詰める**
  - 却下理由: 18,000作品 × メタデータは context window に入らない。詰めても "lost in the middle" で精度が落ちる
- **案C: 手書きで30作品を厳選し、その中だけで推薦させる（現在の `data/works.yml`）**
  - 却下理由: スケールしない。タグ付けの粒度合わせ（「死生観」と「生と死」など）も人手で持続不可能。ただし「初手フォールバック」として残す価値はあるため、廃止せず併用する
- **案D: 本文を取得してLLMに読ませて推薦させる**
  - 却下理由: 本文の取り扱いは README の「やらないこと」に明示的に反する。著作権上は問題ないが、青空文庫へのトラフィック配慮とプロジェクトのスコープ外

## Consequences

### 良いこと
- LLMが知らないマイナー作品にも光が当たる（リスト経由で候補に上がりうる）
- 幻覚（実在しない作品の推薦・URL捏造）をリストで殺せる
- 手書きタグのメンテコストがゼロになる

### 悪いこと / 制約
- 青空文庫CSVの取得・整形パイプラインを持つ必要がある（運用負荷が増える）
- LLM が知らない作品については「リストには載っているが推薦されにくい」という二次的な偏りが残る（ただし案Aよりはマシ）
- CSV の構造変更に追随する必要がある

### 今後の縛り
- 推薦経路を新設する場合も、必ずこのリストを通す（リスト外の出力を許さない）
- `data/works.yml`（厳選30作品）は廃止せず、初手フォールバックとして併存させる
