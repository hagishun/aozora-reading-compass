## 今日決まったこと（3つのADR要点）

### ADR-0001: 推薦は実在リストでgrounding
- LLM単独の推薦は有名作偏重 + 幻覚で失敗する
- 青空文庫CSV（全件）を実在リストとして照合に使う
- works.yml(27件) は初手フォールバックとして併存

### ADR-0002: 推薦経路はサーバーレス関数経由のLLM
- BYOK（フロント直叩き）と事前生成型（Actionsバッチ）を却下
- 理由: LinkedIn訪問者にAI体験を即座に届ける + 「サーバーレス×LLM×公開URL」の地雷を踏む学びが濃い
- リスト照合・rate limit・graceful degradation はすべてWorker側

## 外出先で考えると良い問い

ぼんやり考える系。決まったら次セッションで反映する。

1. **state ラベルの日本語表記**: `recovery / thinking / stimulus / quiet` を画面では何と出す? 
   - 案: 「回復したい / 考えたい / 刺激がほしい / 静けさがほしい」
   - 案: 「やすらぎ / 思索 / 刺激 / しずけさ」
   - 名詞か動詞か、ですます調か、で印象が変わる

2. **サイトのトーン**: フォント・色・余白
   - 青空文庫の「資料感」に寄せる? それとも本屋の「発見感」に寄せる?
   - ダーク/ライト、明朝/ゴシック

3. **「今日の一冊」は1日固定 or 毎クリック更新?**
   - 1日固定 → 占い的な体験、シェアしやすい
   - 毎クリック更新 → 探索的、刺さらなければやり直せる
   - ハイブリッド: 「今日のおすすめ」+「もう一冊」ボタン

4. **`reason` 文の長さ**: LLMに何文字くらい書かせる?
   - 短い (30字): タイトル横に添える
   - 中くらい (100字): 結果ページの本文
   - 長い (300字): 別途「なぜこれ?」展開

5. **LinkedIn 投稿で何を一番訴求するか**
   - 「無料枠だけでAI×OSSを動かした」?
   - 「LLMの幻覚をリスト照合で殺す素朴RAGの実例」?
   - 「青空文庫という文化資産への発見導線」?
   - 訴求軸でサイトのコピーも変わる
