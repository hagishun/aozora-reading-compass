## 発見: Cloudflare の無料枠が想定以上に厚い

ADR-0003 で「Cloudflare に寄せる」と決めた時点では Pages + Workers + Workers AI + KV だけ想定していたが、実際は **クレカ登録なしの Free Plan で総合プラットフォームが揃う**ことが分かった。AWS 同等構成より圧倒的に安く（というか $0）、free-tier-first 原則が当初想定より深く効く。

### Free Plan に含まれる主な無料枠

| サービス | 無料枠 | 用途 |
|---|---|---|
| Pages | 無制限req/帯域、500ビルド/月 | Astro 配信 |
| Workers | 100,000 req/日、CPU 10ms/req | API |
| Workers AI | 10,000 Neurons/日 | LLM呼び出し |
| KV | read 100k/日, write 1k/日, 1GB | rate limit, キャッシュ |
| D1 | 5GB, read 5M/日, write 100k/日 | SQLite (青空文庫CSV格納候補) |
| R2 | 10GB, **egress無料** | 大容量ファイル |
| Queues | 100万msg/月 | 非同期処理 |
| Cron Triggers | Workers枠内 | 定期実行 (今日の一冊バッチ生成) |
| Vectorize | 5M vectorクエリ/月, 30M保存 | semantic search (気分→作品) |
| Web Analytics | 完全無料 | 訪問計測 (LinkedIn公開時) |
| Email Routing | 完全無料 | お問い合わせ受信 |

### このプロジェクトでの追加採用候補（Phase 2以降）

- **Cron Triggers + KV キャッシュ**: 「今日の一冊」を1日1回バッチでLLM生成→KV格納。アクセス時はLLM呼ばずKV読むだけ。Workers AI Neuron節約 + 高速化
- **D1**: ADR-0001 の L2層 (青空文庫CSV全件) を SQLite として格納。Worker から SQL で検索
- **Vectorize**: 自由記述の気分文を embedding 化して、L2の作品メタと semantic 検索でマッチング (気分の言語化が苦手な人向け)
- **Web Analytics**: LinkedIn 経由のリファラ計測。GoogleAnalytics より軽量・プライバシー配慮で プロジェクト主題と整合

### 注意点

- Workers AI のモデル別 Neurons 消費は要チェック（順次有料化されるモデルもある）
- KV の write が一番先に詰まる枠 (1k/日)。rate limit ストアは set 頻度低めに設計
- ロックインは深まる（脱出時は全サービス移植）が、ADR-0003 の Consequences で受容済み
