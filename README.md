# Aozora Reading Compass

> 青空文庫の膨大な作品の中から、「今日の自分に合う一冊」に出会うための小さなOSS実験。

青空文庫は日本語の文化資産として大きな価値を持っていますが、いざ読もうとすると「今日は何を読めばいいか」で止まってしまいがちです。本プロジェクトは、本文を自前で配信するのではなく **発見導線** と **読後ログ設計** だけを担い、AIと制約付き推薦で「今日の一冊」を提案します。

## 解決したいこと

- 青空文庫は膨大だが、**今日の自分に合う一冊にたどり着きにくい**
- 人気順や作者別だけでは、今の自分の状態（疲れている / 考えたい / 刺激がほしい）と接続しない
- 読んだ記録が残らないので、**積み上がっている感覚**が得られない

## やらないこと

このプロジェクトのスコープを小さく保つために、以下は **意図的にやりません**:

- 青空文庫本文の自前ホスティング・自前表示（リンクのみ）
- 課金・サブスク・アカウント機能
- 大規模なWebアプリ化（モバイルアプリ、SNS機能など）
- 全作品を網羅すること（初期は30作品程度の精選）

## 制約付き推薦の考え方

「あなたに合う本」を全自由度で探すと選べなくなるので、3つの軸で **制約** をかけます。

| 軸 | 例 | 役割 |
|---|---|---|
| **状態 (state)** | 回復 / 思考 / 刺激 / 静けさ | 今日の気分・コンディションに合わせる |
| **除外 (exclude)** | 読了済み作品 | 積み上がりを可視化する |
| **テーマ (theme)** | 自然 / 都市 / 人間関係 / 旅 / 死生観 | 縦の繋がりを作る |

これらを掛け合わせて「今日の制約条件で読める一冊」を出します。網羅性より **選択コストの低さ** を優先します。

## 初期30作品の選定基準

- **短編または短い中編**を中心に（10〜40分で読み切れる）
- **作者・テーマ・トーンが偏らない**ように分散
- **入門として有名すぎず、初手として手に取りやすい**もの
- 青空文庫上で**確実にアクセスできる**もの（リンク切れがないこと）

具体的なリストは [`data/works.yml`](data/works.yml) を参照してください。

## 読後ログの形式

読書を「点」ではなく「線」にするために、1作品1行の軽量ログを推奨します。詳細テンプレートは [`templates/reading-log-template.md`](templates/reading-log-template.md)。

例:

```markdown
- 2026-05-03 / 夢十夜 (夏目漱石) / state: 静けさ / 一行: 第一夜の「百年待っていてください」が今日の自分に効いた
```

書き溜めるほど **自分の読み方の傾向** が見えるように設計しています。

## 青空文庫へのリンク方針と著作権への配慮

- 本リポジトリでは **青空文庫の本文をコピーして再配布することはしません**。`data/works.yml` には作品メタデータ（作者・タイトル・タグ・要約可否・青空文庫URL）のみを保持します。
- 本文へのアクセスは **青空文庫の公式URLへのリンク** で完結させます。
- 掲載作品はすべて **著作権保護期間が満了** している（青空文庫公開済み）ものに限定します。
- 青空文庫の運営に対しては、**読者を青空文庫へ誘導する立場** を取り、トラフィックを奪わない形で連携します。

## ステータス

`idea → draft`

このリポジトリは構想段階の実験プロジェクトです。READMEと最小データのみが存在し、コード・UIはまだありません。「AIとOSSで文化資産へのアクセスを少しだけ滑らかにする」という問いへの応答として、ゆっくり育てていきます。

設計の全体像と実装順序は以下を参照:

- [`docs/architecture.md`](docs/architecture.md) — システム構成・データフロー・無料枠の境界
- [`docs/roadmap.md`](docs/roadmap.md) — Phase 0〜5+ の実装順序
- [`docs/adr/INDEX.md`](docs/adr/INDEX.md) — 個別の意思決定記録
